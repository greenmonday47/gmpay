GMPay Checkout Documentation

In order to Proceed you need to have Valid API Keys from your Merchant Profile, if you don't have a merchant account you must register for one before you proceed and further.
Prerequisite
  • Active GMPay Account
  • Verified GMPay Merchant Account
  • API Keys
  • Some Developer Knowledge
Methods
  • POST
  • GET
URL Links
  • https://api.gmpayapp.com/api/v2/transactions/init
  • https://api.gmpayapp.com/api/v2/transactions/process
  • https://api.gmpayapp.com/api/v2/transactions/status/:trx
Initializing Checkout
Before you proceed to take any action you are required to first initialize the checkout page.
https://api.gmpayapp.com/api/v2/transactions/init?phone=PHONE_NUMBER&amount=AMOUNT&merchant=API_KEY

PHONE : STRING | REQUIRED (MUST START WITH COUNTRY CODE OR WITHOUT THE LEADING 0)
AMOUNT : DOUBLE | REQURIED
MERCHANT : STRING | REQUIRED (API KEY CAN BE FOUND IN THE GMPAY APP)

Debit User

URL: https://api.gmpayapp.com/api/v2/transactions/process
Method: POST
Payload

{
"service" : "Checkout",
"amount" : (Number Required),
"account" : (String Required | Must start with country code or actual number without leading 0),
"pubKey" : (String Required | merchant public key)
}

Success Response

{
"status": (HTTP STATUS CODE),
"statusText": (HTTP STATUS NAME),
"data" : {
"reference" : (TRANSACTION ID),
"message": (HUMAN READABLE MESSAGE),
"amount": (ORIGINAL AMOUNT),
"username": (USER FULLNAME),
"phoneNumber": (USER PHONE NUMBER),
  }
}

Error Response

{
"name" :"APIClientError",
"status" : (HTTP STATUS CODE),
"statusText" : (HTTP STATUS NAME),
"error" : 
   {
"message" : (HUMAN READABLE ERROR)
   }
}

Callback Response

{
status : (TRANSACTION STATUS | AVAILABLE STATUS : timeout, success, failed, canceled),
transacionId : (TRANSACTION ID FROM GMPAY API),
amount : (TRANSACTION ORIGINAL AMOUNT),
username : (USER FULLNAME),
phoneNumber : (USER PHONE NUMBER),
trxStatus : (ACTUAL TRANSACTION STATUS VIEWABLE IN GMPAY APP, COULD BE DIFFERENT FROM STATUS ABOVE)
}
Automatic Withdrawals to GMPay users account

URL: 'https://api.gmpayapp.com/api/v2/transactions/status/:trx'
Method: GET
trx : (TRANSACTION ID)
Payload

{
"status": (HTTP STATUS CODE),
"statusText": (HTTP STATUS NAME),
"data" : {
"id" : (TRANSACTION ID),
"status": (success | failed | canceled | pending | initiated),
"amount": (ORIGINAL AMOUNT),
"createdAt": (DATE),
"updatedAt": (DATE),
  }
}
